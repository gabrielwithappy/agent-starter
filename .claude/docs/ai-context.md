# AI Context 문서 관리 가이드

AI Context 문서는 Claude가 프로젝트/도메인별 배경 지식을 참고할 수 있도록 작성하는 문서입니다.

## 명명 규칙

**형식**: `(Context) [주제명].md`

**예시**:
- `(Context) 영신식품.md`
- `(Context) React 개발.md`
- `(Context) 마케팅 캠페인.md`

**검색**: Obsidian에서 `(Context)` 검색으로 모든 Context 문서 찾기

## 생성 방법

### 1. 템플릿 사용
`.claude/templates/(Context) AI Context Template.md` 템플릿을 복사

### 2. 적절한 위치에 배치
- 프로젝트 관련: `obsidianKMS/01.Projects/`
- 영역 관련: `obsidianKMS/02.Areas/`
- 참고 자료: `obsidianKMS/03.Resource/`

### 3. Frontmatter 작성
```yaml
---
created: 2026-02-14
tags:
  - context
  - [도메인]
status: active
review-cycle: monthly
---
```

### 4. 점진적 업데이트
업무 중 발견한 정보를 지속적으로 추가

## 문서 구조

### 핵심 섹션

1. **프로젝트/도메인 개요**
   - 목적과 범위
   - 주요 이해관계자

2. **핵심 개념**
   - 용어 정의
   - 비즈니스 규칙
   - 제약사항

3. **반복 패턴**
   - 자주 사용하는 워크플로우
   - 일반적인 해결책

4. **발견 및 학습 내용**
   - 최근 발견한 정보
   - 문제 해결 경험
   - **크기 관리**: 최근 10-20개 항목만 유지

## 유지보수

### 정보 생애주기

```
발견 → 검증 → 핵심 개념으로 승격
```

1. **발견**: "발견 및 학습 내용"에 임시 기록
2. **검증**: 반복 사용/확인되면 검증 완료
3. **승격**: 검증된 정보는 "핵심 개념" 또는 "반복 패턴"으로 이동

### 정기 검토

- **주기**: frontmatter의 `review-cycle` 참조
  - `weekly`: 매주
  - `monthly`: 매월
  - `quarterly`: 분기별

- **검토 항목**:
  - 오래된 정보 삭제
  - 중복 내용 통합
  - 새로운 패턴 식별

### 크기 관리

- **목표**: 문서당 200-300줄 이내
- **방법**:
  - 오래된 "발견 및 학습 내용" 아카이빙
  - 세부 내용은 별도 문서로 분리
  - Context 문서는 "요약 + 링크" 역할

## 활용 시나리오

### 시나리오 1: 새 프로젝트 시작
1. 템플릿으로 Context 문서 생성
2. 초기 요구사항 기록
3. 작업하며 발견사항 추가

### 시나리오 2: 기존 프로젝트 재개
1. Context 문서 검색
2. Claude에게 Context 문서 경로 제공
3. Claude가 배경 지식 기반으로 작업

### 시나리오 3: 지식 공유
1. Context 문서를 팀원과 공유
2. 동일한 배경 지식으로 협업
3. 새로운 발견사항 공동 업데이트
