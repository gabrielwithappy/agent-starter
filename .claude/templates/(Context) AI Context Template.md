## 💡 템플릿 사용 가이드 (완성 후 삭제)

**사용 방법**:
1. 템플릿 복사 후 기본 정보만 작성
2. 업무 중 발견할 때마다 해당 섹션에 추가
3. 날짜와 출처 반드시 기록
4. 불필요한 섹션은 삭제

**섹션 활용**:
- **핵심 개념 및 제약사항**: 새로운 용어/개념 발견 시, 시스템 동작 원리 이해했을 때, 제약/이슈 부딪힐 때 (🔴 필수 / 🟡 권장 구분)
- **반복 패턴**: 같은 작업 2회 이상 반복 시
- **발견 내용**: 중요한 인사이트 발견 시 (최근 10-20개만 유지)
- **참고 자료**: 외부 문서, 도구, 내부 문서, 연결된 Context

**정보 생애주기** (Context 크기 관리):
1. **발견**: "📝 발견 및 학습 내용"에 최초 기록
2. **검증**: 2-3회 이상 활용 확인되면 "💡 핵심 개념 및 제약사항"으로 승격
3. **패턴화**: 동일한 방법이 반복되면 "🔄 반복 패턴"으로 승격
4. **정리**: 6개월 이상 미사용 항목은 주기적으로 삭제 또는 아카이브
5. **승격 후 원본 삭제**: 승격된 정보는 원래 섹션에서 제거하여 중복 방지

**중복 방지 규칙**:
- 동일한 정보가 여러 섹션에 중복 기록되지 않도록 주의
- 새 정보 추가 전 기존 내용 검색 (Ctrl+F)
- 유사한 내용은 하나로 통합하고 출처만 추가

**원칙**:
- ✅ 작은 정보부터 시작
- ✅ 출처와 날짜 필수 기록
- ✅ 주기적 검증 및 업데이트 (review-cycle 준수)
- ✅ 정보 승격으로 Context 크기 관리
- ❌ 한 번에 완성하려 하지 않기
- ❌ 추측성 정보 기록 금지
- ❌ 동일 정보 중복 기록 금지
